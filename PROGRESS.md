| OB-1 hardening (round 2) | ✅ done | A second read-only audit found 4 more gaps, all fixed with structural kills (heuristics leave residue): **(P1)** the credential guard was a keyword denylist (`webhook_secret`/`code`/`#access_token` slipped past) — replaced with a **structural URL-shape rule**: a `callback_url` is a bare scheme+host+path; userinfo, ANY query string, and ANY fragment are rejected (the three places a secret hides), so the class is closed regardless of param name. Adapters route via `correlation_id`, authenticate via `X-Atlas-Signature`. **(P1)** reconcile racing live completion created two rows with different ids (receiver couldn't dedupe) — the automatic completion delivery now uses a **deterministic id** (`_completion_delivery_id(run_id)`) inserted with `INSERT OR IGNORE` (`db.claim_delivery` returns a `created` flag); only the winner drives the send, so a run has exactly one completion `delivery_id`. **(P2)** `ATLAS_OUTBOUND_TIMEOUT` only bounded the body drain, not the status/header read (a slow-trickle header held a thread ~indefinitely) — added a **total wall-clock watchdog** in `_send` that `shutdown(SHUT_RDWR)`s the socket at the deadline (a plain `close()` won't interrupt an in-flight `getresponse()`; the `HTTPResponse` fp keeps the fd open) and treats a response salvaged after the deadline as a `TimeoutError` (a slow *body*, status already read, still succeeds). **(P2)** reconcile scanned only the newest 10k runs — now complete via `db.iter_pending_delivery_ids()` (indexed on status, all ages) + `db.runs_missing_webhook_delivery()` (`json_extract` anti-join, only webhook runs missing a row). `check_outbound.py` + `check_input_adapter.py` extended (structural shape rejection incl. a keyword-denylist-evading param, deterministic-id convergence, slow-header total-deadline bound, complete reconcile); gate + lint green. |
| OB-1 hardening (round 1) | ✅ done | A read-only audit found 5 gaps, all fixed: **(P1)** `OutboundService.reconcile()` — restart recovery for a `pending` delivery whose attempt thread died, or a completed run that crashed before its delivery row was ever written; run in a background thread from `AtlasRuntime.__init__` (mirrors `reconcile_runs`/`reconcile_jobs`). **(P1)** `callback_url` embedding credentials (userinfo or a credential-shaped query key like `token`/`secret`/`api_key`) is rejected in `resolve_outbound_target` — never persisted into run input / the deliveries ledger, never returned via read APIs; adapters authenticate via `X-Atlas-Signature` instead. **(P2)** an exact-hostname allowlist match now always resolves DNS too (previously skipped resolution entirely), so the sender always pins the actual connection to a validated address, closing a rebinding gap for operator-trusted hostnames. **(P2)** response draining is bounded by bytes AND a wall-clock deadline using `read1()` (not `read()`, which loops internally trying to fill the buffer — same pitfall already documented on `iter_sse` in `thclaws_client.py`); a receiver that trickles bytes can no longer hold a delivery attempt open indefinitely. **(P3)** `delivery.blocked`/`delivery.delivered`/`delivery.failed` audit entries added (previously only `delivery.create` existed). `scripts/check_outbound.py` + `scripts/check_input_adapter.py` extended (credential rejection, reconcile, bounded-drain timing, audit entries); gate + lint still green. |
