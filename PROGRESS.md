## Security findings

No open findings. Everything codex surfaced across the run was fixed and guarded by a hermetic check (see per-stage rows). Two intentional exceptions to per-user auth exist, both documented in `docs/specs/threat-model.md`: `GET /healthz` (unauthenticated, returns only `{ok, service, version}`) and `POST /api/worker-callbacks/{job_id}` (T3 pre-auth carve-out, authorized by its own per-dispatch HMAC token with body cap, idempotent apply, and system audit actor).
